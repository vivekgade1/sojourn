# Sojourn V2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** V2 closes the loop V1 opened: flags become storm-proof and reach the terminal, restores become exact and *reversible* (harvest back), the graph becomes queryable by humans and agents, and the platform gains the trust infrastructure (migrations, GC, secrets hardening, precision benchmark) that everything else stands on.

**Architecture:** Same monorepo, same principles (§2 of V1 spec) plus one V2 principle: **user-facing actions never queue behind capture** (proven pattern: private-index safety snapshots). All schema changes flow through a new migration framework. All new checks/behaviors get E2E scenarios in `scripts/e2e`.

**Tech stack additions:** `@modelcontextprotocol/sdk` (soj mcp), `esbuild` (devDep, plugin bundling). Nothing else.

## Global Constraints

- TypeScript strict ESM NodeNext; relative imports carry `.js`; core barrel is controller-owned.
- Tests never touch the network or real `~/.sojourn`/`~/.claude` (inject fetchJson; tmp SOJOURN_HOME/CLAUDE_CONFIG_DIR).
- Verified flags stay precision-first; **digest/rollup flags must never hide the flagship** (edit_claim_mismatch gets the largest budget) and never blur verified/advisory.
- Restore/harvest/rewind: safety snapshot first, freshness validation, never write the user's `.git` object store, abort-clean on conflict (no partial applies without markers + explicit mode).
- Exact rewind writes ONLY new files into the Claude projects dir (never mutates existing transcripts) and REFUSES (clear message) when the ancestor chain crosses a compaction/summary boundary.
- Every V2 feature lands with: unit tests + an E2E cycle scenario (Task 14) where observable via API.
- Schema changes ONLY via the Task 1 migration framework (`PRAGMA user_version`), with an upgrade test from a real V1 database fixture.
- No npm publish, no launchd install, no live-OpenCode dependency anywhere.

## Contracts (Task 0, controller — written before any wave)

New types in `packages/core/src/types.ts` (verbatim):

```ts
export interface StoredFlag { /* + */ suppressedCount?: number; }   // >0 ⇒ digest flag: evidence is a sample, count covers identical-claim siblings
export interface SessionHealth {
  sessionId: string; turns: number; verifiedActive: number; verifiedResolved: number;
  advisoryActive: number; dismissed: number; suppressed: number;
}
export interface HarvestPreflight {
  worktreePath: string; originNodeId: string; baseTree: string; branchTree: string;
  files: Array<{ path: string; status: "clean" | "conflict" | "identical" }>;
  mainlineDirty: boolean; warnings: string[];
}
export interface HarvestResult {
  applied: string[]; conflicted: string[];            // conflict-marked files written
  skippedIdentical: string[]; safetySnapshotRef: string; patchPath: string | null; mergeNodeId: string | null;
}
export interface RewindPlan {
  mode: "exact" | "tip";                               // exact = synthesized transcript
  newSessionId: string | null; transcriptPath: string | null;
  refusedReason: string | null;                        // e.g. "chain crosses compaction boundary"
  resumeCommand: string;
}
export interface SearchHit { node: ChronoNode; score: number; snippet: string; }
```

Schema migration v2 (Task 1 executes via framework): `flags` + `suppressed_count INTEGER NOT NULL DEFAULT 0`; new table `node_files(node_id TEXT, path TEXT, status TEXT)` (+ index on path); FTS5 virtual table `node_fts(node_id UNINDEXED, content)` populated on upsert (summary + label + annotations text).

API additions (append to docs/API.md in Task 0):
```
GET  /api/sessions/:id/health          -> SessionHealth
GET  /api/search?projectId&q&file      -> { hits: SearchHit[] }        (FTS + node_files)
POST /api/nodes/:id/rewind-plan        -> RewindPlan                    (no side effects)
POST /api/nodes/:id/rewind             -> RewindPlan (executed: transcript written when mode=exact)
POST /api/worktrees/harvest/preflight  { worktreePath } -> HarvestPreflight
POST /api/worktrees/harvest            { worktreePath, mode: "apply" | "patch" } -> HarvestResult
GET  /api/sessions/:id/turn-flags?sinceNodeId -> { lines: string[] }    (compact, budgeted, verified-only)
```

## Task capsules (briefs — exact behavior each implementer must satisfy)

### Task 1: Migration framework + schema v2
`runMigrations(db)` on GraphStore open: `PRAGMA user_version` gates ordered migration list; v0/v1→2 adds `flags.suppressed_count`, `node_files`, `node_fts` (FTS5, contentless ok). Populate FTS/node_files going forward in `upsertNode`/`setSnapshotRef` callers? NO — population stays in Task 8; this task ships empty structures + framework + tests: fresh db lands at version 2; a checked-in V1 fixture db upgrades losslessly (row counts preserved); reopening is idempotent; failed migration rolls back (transaction) and throws typed `SojournMigrationError`.

### Task 2: Flag budgets, rollups, digests, health
`core/src/flags/budget.ts`: `applyBudgets(flags: Flag[], opts): { kept: Flag[]; digests: Flag[]; suppressed: number }` — per-kind per-turn budget (default: edit_claim_mismatch 10, others 3; configurable), identical-claim dedup (reuse `flagsMatchSameClaim` tokens), overflow → ONE digest flag per kind (evidence = first sample + "…and N similar suppressed", carries suppressedCount). Engine/ingest integration point: budgets run over a TURN's freshly produced flags before storage; store.addFlag persists suppressedCount. `getSessionHealth(store, sessionId): SessionHealth` counts from flags tables (pure counts — no scoring). Tests: overflow collapses with flagship surviving longest; TN: under-budget unchanged; digest never mixes kinds/tiers; health counts exact.

### Task 3: Retention/GC + secrets hardening
`gcShadowRepo(snapshotter, opts {keepDays, pins: Set<tree>, dryRun})`: pins = every snapshotRef referenced by nodes with kind checkpoint/decision/assumption, flagged nodes, restore manifests, plus refs/sojourn/safety history; write `refs/sojourn/keep/<n>` for pinned commits reachable-selection, then `reflog expire --expire=now` + `gc --prune=now` in SHADOW repo only. Archive-before-prune: `git bundle create <archive>` of pruned range when `opts.archiveDir`. Dry-run default returns {wouldPruneObjects, reclaimableBytes-est}. Store marks pruned refs: `markSnapshotPruned(tree)` → preflight refuses with "thinned by retention policy". Secrets: extend `EXCLUDE_ENTRIES` (+`*.p12 *.keystore id_rsa* *.token credentials* .aws/ .ssh/ *.secret`), test that a planted `.env`/`id_rsa` never appears in `listFiles`. `soj gc [--days N] [--run]` CLI.

### Task 4: Edit-claim precision benchmark
`scripts/bench/editclaim-corpus.jsonl`: ≥40 labeled cases {text, diffPaths, expect: flag|silent, note} — include the session's historical traps (hedged claims, quoted requirements, "once tests pass", path aliases, bare filenames, multi-claim). `scripts/bench/run.mjs` runs extractEditClaims+editClaimCheck against each, prints precision/recall; `packages/core/test/flags/bench.test.ts` asserts precision == 1.0 and recall >= 0.8 over the corpus (corpus is version-controlled ground truth).

### Task 5: Exact-node rewind (transcript synthesis)
`adapter-claude/src/rewind.ts`: `planRewind(nodes, targetNodeId, projectsDir)` → RewindPlan: walk ancestor chain via parentId (cycle-guarded); REFUSE exact mode (mode:"tip", refusedReason) when: any ancestor is missing from the set (orphan gap), or the ORIGINAL transcript contains `type:"summary"`/`isCompactSummary`/sidechain markers within the chain's uuid range (compaction honesty — pass raw line index in). `executeRewind(plan, rawLines)` writes NEW `<newSessionId>.jsonl` (uuid-remapped ancestor lines, sessionId rewritten, order preserved, original untouched) into the SAME projects subdir; returns resumeCommand `claude --resume <newSessionId>` (exact) or `claude --resume <orig> --fork-session` (tip). Restore engine/daemon: `/api/nodes/:id/rewind-plan` (pure) + `/api/nodes/:id/rewind` (writes); restore route response gains `rewind` field; UI untouched (V2.1). Tests: chain synthesis round-trips through parseSessionJsonl with identical kinds/order; refusal cases; original file byte-identical after execute; uuid collision-free.

### Task 6: Harvest (the return path)
`core/src/harvest/harvestEngine.ts`: `preflight(worktreePath)` reads `.sojourn-restore.json` (typed error if absent), baseTree = manifest.treeHash; branchTree = snapshotSafety() ON THE WORKTREE (constructor with projectRoot=worktree, same shadow repo); per-file three-way classify via `git merge-file` dry runs: identical | clean | conflict. `harvest(worktreePath, mode)`: safety snapshot of MAINLINE first; mode "apply": write merged contents (conflict markers included for conflicted files when `allowConflicts`, else abort-clean listing them) into mainline working tree (fs writes only — never git plumbing against user repo); mode "patch": emit `git diff baseTree..branchTree` to `<worktree>/.sojourn-harvest.patch`, write nothing to mainline. Graph: insert `fork_point`-closing node kind "checkpoint" labeled `harvest: <n> files` parented to origin node (store dep injected). Tests with real snapshotter+tmp dirs: clean apply lands edits; conflicting mainline edit → abort-clean (no partial writes) unless allowConflicts (markers written); patch mode writes patch only; safety snapshot always precedes writes.

### Task 7: Worktree project aliasing
Daemon ingest: when a batch's project root contains `.sojourn-restore.json`, resolve origin project (manifest nodeId → store lookup) and ingest under the ORIGIN projectId (sessions tagged: SessionRow gains no schema change — label session title `worktree:<node8>` via upsertSession title). Nodes get `meta.forkedFrom = manifest.nodeId` and the first ingested node parents to manifest.nodeId (graph edge from branch to its origin!). Test: ingest batch from a fake worktree with manifest → nodes appear under origin project, first node's parentId = origin node id.

### Task 8: Decision memory (FTS + files index + search API + CLI)
Store: populate `node_fts` on upsertNode (summary+label) and on addAnnotation; populate `node_files` in ingest where per-node turn diffs are computed (daemon passes FileChange[] per node batch tail — index the TURN's files onto its prompt node id + batch-last node). `search(projectId, {q?, file?, kinds?}): SearchHit[]` — FTS MATCH + join, bm25 score, snippet(). Daemon `GET /api/search`. CLI: `soj why "<q>" [--project]` (hits with kind-colored labels, node ids, gists), `soj decisions [--file P]` (marks + flagged nodes filtered). Tests: FTS finds annotation text; file query returns turns touching path; kinds filter; empty q + file works.

### Task 9: `soj mcp` — local MCP server
`packages/cli/src/mcp.ts` using `@modelcontextprotocol/sdk` stdio server exposing READ-ONLY tools: `sojourn_search {query, file?}`, `sojourn_decisions {project?}`, `sojourn_flags {sessionId?}`, `sojourn_node {nodeId}` — each calls the daemon HTTP API (never the DB directly; daemon may be down → tool returns friendly error text). `soj mcp` runs it; README section: how to add to Claude Code (`claude mcp add sojourn -- soj mcp`). Tests: spawn server over stdio pipes, initialize handshake, tools/list, one tools/call against a stub daemon.

### Task 10: Terminal flag delivery
Daemon: `GET /api/sessions/:id/turn-flags?sinceNodeId=` → last turn's ACTIVE verified flags → compact single-line strings (budgeted via Task 2, max 3 lines + "+n more"). Hook (`postToolUse.ts`): only when `SOJOURN_HOOK_FLAGS=1` AND event is Stop: after the existing fire-and-forget POST, GET turn-flags (same 500ms budget) and print lines to stdout (Claude Code shows hook stdout to the user); never blocks, always exit 0, silent on any error. Tests: hook prints when env set + server responds; stays silent without env; exit 0 with server down.

### Task 11: `soj gate`
`soj gate [--session <id>|--project <id>] [--include-advisory]`: fetch flags via API; unresolved active verified flags ⇒ exit 2 with a table + evidence; clean ⇒ exit 0 "gate passed: N turns, 0 active verified flags"; daemon unreachable ⇒ exit 3 (distinct). Honest header ALWAYS states what was checked ("claims vs snapshots recorded by the local daemon"). Tests: stub server both outcomes + exit codes.

### Task 12: Packaging (publish-ready, no publish)
esbuild (devDep, root) bundles `adapter-claude/src/hooks/postToolUse.ts` → `plugins/claude/hooks/sojourn-hook.mjs` (self-contained, node shebang); hooks.json points at `"${CLAUDE_PLUGIN_ROOT}/hooks/sojourn-hook.mjs"` (no repo-relative escape); build script `npm run build:plugin`. Publish-readiness: cli/daemon/core/adapters get `files`, `publishConfig.access=public` REMOVED/left private:false? — set real names unscoped later; keep `private:true` at root only; document `npm pack` output sanity in README-dev. Plugin validate passes. Tests: built hook runs standalone (spawn with stub server), hooks.json path resolves inside plugin dir.

### Task 13: Conformance fixtures

### Task 14: E2E cycle V2 scenarios
Extend gen-session/api-check: flag-storm turn (15 identical false claims → expect ≤ budget kept + 1 digest with suppressedCount), `GET health` counts, `search?q=` + `file=` hits, rewind-plan exact on clean chain + REFUSED on synthetic compaction session, harvest round-trip (restore → mutate worktree → preflight → apply → mainline updated + safety snapshot + merge node; conflict case → abort-clean), aliasing (ingest worktree session → origin project), `soj gate` exit codes live, turn-flags route budget cap.
