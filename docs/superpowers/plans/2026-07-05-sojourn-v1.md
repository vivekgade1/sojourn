# Sojourn V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement Sojourn V1 per `SOJOURN_BUILD_PLAN_V1.md` — a daemon + adapters + web UI that captures Claude Code (and OpenCode) sessions into a persistent cross-session decision graph with whole-tree snapshots, deterministic T1 confidence flags, restore-to-worktree, a React Flow UI, and a `soj` CLI.

**Architecture:** TypeScript npm-workspaces monorepo. CLI-agnostic core (graph store, snapshotter, flag engine, restore engine) + per-CLI adapters (ingestion + conversation-restore) + Node daemon (HTTP/WS + watchers) + Vite/React Flow web UI + commander CLI. Shared contracts live in `packages/core/src/types.ts` and `packages/core/src/interfaces.ts` and are written BEFORE any parallel work begins.

**Tech Stack:** Node ≥20, TypeScript (strict, ESM), better-sqlite3, system `git` via execFile (behind an interface), express + ws, chokidar, commander, React + reactflow + dagre, Vite, vitest.

## Global Constraints

- TypeScript `strict: true`, `"type": "module"` everywhere; Node ≥ 20.
- Runtime home: `~/.sojourn` — overridable via `SOJOURN_HOME` (tests MUST set a temp `SOJOURN_HOME`).
- DB at `$SOJOURN_HOME/sojourn.db`; shadow repos at `$SOJOURN_HOME/snapshots/<projectId>/`; worktrees at `$SOJOURN_HOME/worktrees/`.
- NEVER touch the user's `.git`. Shadow git only, driven by `GIT_DIR`/`GIT_WORK_TREE`/`GIT_INDEX_FILE` env.
- Every restore path: safety snapshot FIRST, then `hasTree()` freshness validation, then checkout into a NEW worktree dir. Never restore in place in v1.
- T1 flags are `tier: "verified"`, deterministic only, always carry `evidence`; tune precision over recall (a missed flag is fine; a false flag is not). T2 critic is `tier: "advisory"`, OFF by default, requires explicit opt-in + user-supplied API key.
- Ingestion must never throw out of the watcher loop — catch, log to stderr, continue.
- No network in unit tests. Registry lookups go through an injected `fetchJson` so tests mock them.
- Node IDs: `"<cli>:<nativeUuid>"`. Project IDs: first 12 hex chars of sha256 of the resolved project root path. Ingestion is idempotent (upserts).
- Daemon port 4177 (env `SOJOURN_PORT`).
- Keep all parallel-tool-call siblings in the tree (do not replicate Claude's sibling-drop bug).

## Task 2: `core/snapshot` — ShadowSnapshotter

**Files:** Create `packages/core/src/snapshot/shadowSnapshotter.ts`, `packages/core/src/snapshot/git.ts` (execFile wrapper), `packages/core/src/snapshot/index.ts`; Test `packages/core/test/snapshotter.test.ts`.

**Interfaces:** Implements `SnapshotterLike` (interfaces.ts) exactly. Constructor: `new ShadowSnapshotter({ projectRoot, shadowDir })`.

Implementation requirements:
- `init()`: `git init` a repo whose gitdir is `shadowDir` (use `git init --separate-git-dir`? NO — simplest: `git --git-dir=<shadowDir> init` then `core.bare=false`); write `<shadowDir>/info/exclude` with: `.git/`, `node_modules/`, `dist/`, `build/`, `out/`, `.next/`, `.cache/`, `*.log`, `.env`, `.env.*`, `*.pem`, `*.key`, `.sojourn/`, `.DS_Store`. Set `user.email=sojourn@local`, `user.name=sojourn` in shadow config.
- All git calls: `execFile("git", args, { env: {...process.env, GIT_DIR: shadowDir, GIT_WORK_TREE: projectRoot, GIT_INDEX_FILE: join(shadowDir,"sojourn-index") } })`.
- `snapshot()`: `add -A` then `write-tree` → tree hash; then `commit-tree <tree> -m snap [-p <prevHead>]` and update a ref `refs/sojourn/head` (keeps objects reachable for GC; cheap).
- `diff(a,b)`: `diff-tree -r --name-status [--root] a b` parse to `FileChange[]` (when `a` is null use `--root` against b's commit or `git ls-tree` as all-A).
- `diffFile(a,b,path)`: `diff-tree -r -p a b -- path` (unified text; empty string if none).
- `hasTree(t)`: `cat-file -t t` === "tree".
- `listFiles(t)`: `ls-tree -r --name-only t`.
- `readFile(t, p)`: `show t:p` (null on missing).
- `restoreToWorktree(t, dest)`: mkdir -p dest; use a TEMP index file: `read-tree t` then `checkout-index -a --prefix=<dest>/` (prefix MUST end with `/`). Never touches projectRoot.

- [ ] TDD in a tmp dir project: snapshot → mutate file → snapshot ⇒ different tree hashes; diff lists M/A/D correctly; excluded dirs (`node_modules/x.js`) never appear in `listFiles`; `restoreToWorktree` reproduces byte-identical content of snapshot 1 in a fresh dest while projectRoot keeps mutated state; `hasTree("deadbeef".repeat(5))` → false; unchanged tree ⇒ identical hash (dedup).
- [ ] Implement; all tests pass; report DONE.

Claude JSONL facts (verify against a real file in `~/.claude/projects/` during implementation): one JSON object per line; relevant line fields: `type` ("user"|"assistant"|"summary"|"system"), `uuid`, `parentUuid`, `sessionId`, `cwd`, `timestamp`, `message` (`{role, content}` where content is string or array of blocks `{type:"text"|"tool_use"|"tool_result", ...}`), `isSidechain`, `toolUseResult`.

Mapping rules:
- user line, string/text content → one `prompt` node (nativeUuid = line uuid).
- assistant line → for EACH content block, one node: text → `assistant` (nativeUuid = uuid or `uuid#i` for i>0), tool_use → `tool_use` (nativeUuid = block `id`), parent chain: first block parents to line's parentUuid-mapped node; subsequent blocks parent to previous block's node.
- user line whose content is tool_result blocks → one `tool_result` node PER block, parented to the `tool_use` node with matching `tool_use_id` (fall back to line parentUuid). KEEP ALL SIBLINGS.
- `summary`/`system`/`isSidechain` lines: skip in v1 (do not crash).
- Malformed lines: skip silently, keep parsing.
- `project.root` = most common `cwd` in file; session id = file's `sessionId` (fallback: basename); `summary` field of node = first 120 chars of text / tool name + first arg.
- Node `id` = `claude:<nativeUuid>`; `parentId` mapped through same rule; null when parentUuid absent/unknown.

## Task 4: `core/flags` — T1 deterministic engine

**Files:** Create in `packages/core/src/flags/`: `engine.ts`, `claims.ts` (claim extraction helpers), `editClaim.ts`, `packages.ts`, `fileRefs.ts`, `symbols.ts`, `tests.ts`, `index.ts`; Test `packages/core/test/flags/*.test.ts` (one file per check + engine).

**Interfaces:** Consumes `CheckContext`/`FlagCheck` from `interfaces.ts`. Produces:

  constructor(checks: FlagCheck[]);                       // default: allT1Checks()
  runOnNode(ctx: CheckContext): Promise<Flag[]>;          // runs applicable checks, concats
}
export function allT1Checks(): FlagCheck[];
export function autoResolveFlags(store: GraphStoreLike, node: ChronoNode, ctx: CheckContext): Promise<number>;
// re-runs T1 checks referenced by earlier ACTIVE flags in the same session; if a previously
// flagged condition no longer holds at this node's tree, resolveFlag(id).

Check specs (ALL: only `appliesTo` assistant-kind nodes with string text content; every produced flag: `tier:"verified"`, `source:"deterministic"`; evidence must name the claim AND the ground truth):

1. `editClaim.ts` — extract claims via regex over assistant text: verb ∈ (updated|edited|modified|changed|created|added|wrote|rewrote|fixed|refactored|renamed|deleted|removed) within 60 chars before a backticked token containing `.` or `/` that looks like a relative file path (reject URLs, commands with spaces, globs). Claim types: EDIT (verbs update/edit/modify/change/fix/refactor/rewrite), CREATE (create/add/write), DELETE (delete/remove). Compare against `ctx.diff` paths (basename match allowed when claim is bare filename and exactly one diff path has that basename). EDIT claim & path ∉ diff ⇒ flag high. CREATE & path ∉ diff-added ⇒ high (M counts as mismatch=medium). DELETE & path ∉ diff-deleted ⇒ high. If `ctx.diff` empty AND ≥1 edit claim ⇒ one high flag per claimed path. If nodeTree/parentTree null ⇒ produce NO flags (no ground truth, stay silent).
2. `packages.ts` — from diff-added lines? v1 simpler + still deterministic: from files listed A/M in `ctx.diff` with ext .ts/.tsx/.js/.jsx/.mjs/.py, read content at `nodeTree` via snapshotter, extract imports (`import ... from "x"`, `require("x")`, python `import x` / `from x import`). Keep bare package names (strip subpaths/scopes correctly: `@s/p/sub`→`@s/p`). Drop: relative paths, `node:*` + Node builtins list, Python stdlib list (embed a reasonable list), names already in any workspace/nearest `package.json` deps at nodeTree, names present under `node_modules` on disk. Remaining candidates → `ctx.fetchJson` npm (`https://registry.npmjs.org/<name>`) for js, PyPI (`https://pypi.org/pypi/<name>/json`) for py. status 404 ⇒ `package_hallucination` high. Network error/other status ⇒ no flag (fail open). Cache lookups per run.
3. `fileRefs.ts` — backticked tokens in text that look like repo-relative file paths (contain `/` AND an extension; reject absolute paths outside root, URLs, globs, `node_modules`). If nodeTree present: flag `file_ref_missing` medium when path ∉ `listFiles(nodeTree)` AND ∉ diff-deleted... only when the sentence implies existence (token preceded within 40 chars by ∈ (in|at|inside|see|check|open|file|defined in|located)). Precision first.
4. `symbols.ts` — pattern: `` `name()` `` or "function/method/class `name`" WITH a co-referenced file (a fileRef in the same sentence resolving to an existing file at nodeTree). Read that file; if `name` does not appear as substring ⇒ `symbol_not_found` high. No co-referenced file ⇒ no flag.
5. `tests.ts` — text matches /\b(all\s+)?tests?\s+(now\s+)?(pass|passing|green)\b/i or /\bbuild\s+(succeeds|passes|is green)\b/i. Scan `ctx.priorNodes` BACKWARD from node to the previous `prompt` node for `tool_use`(Bash-like) + its `tool_result` whose text matches a runner signature (vitest|jest|mocha|pytest|go test|cargo test|npm test|pnpm test|tsc) — and result does NOT match /(fail|failed|failing|error|✗|✖|FAILED|[1-9]\d* failed)/i. Found good run ⇒ no flag. Found failing run ⇒ `test_claim_unverified` HIGH ("claimed tests pass; observed failing run"). No run at all ⇒ `test_claim_unverified` medium ("no test run observed since last prompt").

- [ ] TDD each check: ≥1 true-positive, ≥2 true-negative (precision) cases; engine test: runOnNode concatenation + appliesTo gating; autoResolve test: flag on node A (file not edited), later node B whose diff contains the file ⇒ flag resolved.
